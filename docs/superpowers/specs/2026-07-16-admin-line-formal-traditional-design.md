# Aurora 後台、報價聯絡與繁體書面語設計說明

## 目標

在不重新設計網站、不更改已批准價格規則及 Gemini 設定的前提下，完成以下三項：

1. 整理並驗證現有後台管理功能，讓管理員可自行調整價格、完成時間、公告及服務狀態。
2. 在報價結果加入 WhatsApp 與 LINE 聯絡方式，兩者使用同一份報價內容。
3. 將顧客及管理員可見的預設中文統一為繁體書面語。

SEO、獨立網域、訂單管理、付款管理及客戶資料管理不屬於本次範圍。

## 一、後台管理

沿用現有後台與中央價格資料，不另建第二套價格系統。後台提供：

- 管理員登入。
- 依遊戲及服務調整價格規則。
- 調整預計完成時間、備註及報價有效期。
- 開啟或暫停個別服務。
- 編輯網站公告。
- 顯示尚未儲存的變更，並在儲存前要求確認。
- 儲存後由前台報價表及 Aurora 客服讀取同一份最新資料。

巔峰賽及英雄戰力標等尚未設定正式價格的服務繼續顯示「待人工確認」，不得產生推測金額。

本次不加入訂單列表、付款紀錄、客戶名單或員工權限分級。

## 二、報價結果按鈕

報價結果保留一致的按鈕排列：

1. 複製報價
2. 傳送至 WhatsApp
3. 傳送至 LINE
4. 修改資料

WhatsApp 會開啟指定客服對話並預先帶入完整報價文字。

目前使用的是個人 LINE 加好友連結，並非 LINE 官方帳號，因此不能同時指定收件人及預填訊息。LINE 按鈕採用已批准的替代流程：

1. 使用者按下「傳送至 LINE」。
2. 網站自動複製完整報價內容。
3. 同時開啟現有 LINE 加好友頁面。
4. 網站提示「報價已複製，加入好友後請貼上並傳送」。

如瀏覽器不允許自動複製，網站會顯示清楚提示，讓使用者先按「複製報價」再前往 LINE。LINE 連結不會被當作已成功送出訊息。

日後建立 LINE 官方帳號後，可改用官方帳號直接聊天連結；此升級不屬於本次範圍。

## 三、繁體書面語

繁體中文仍為預設語言。以下顧客及管理員可見內容全部改為自然、正式的繁體書面語：

- 首頁、選單及服務內容。
- 手動報價表及報價結果。
- Aurora 客服問題、回答、提示及錯誤訊息。
- WhatsApp 與 LINE 報價摘要。
- 後台登入、欄位、按鈕、提示及錯誤訊息。

網站不再用粵語口語作為預設顯示文字。例如「幾多錢」改為「多少錢」、「我哋」改為「我們」。

系統仍保留對香港粵語、常見錯字及口語輸入的理解能力；這些辨識詞只用於內部判斷，不會成為預設回覆。顧客即使用粵語提問，Aurora 客服也以繁體書面語回答。

英文及簡體中文選項繼續保留，不改變既有切換方式。

## 四、資料一致性與安全

- 後台、手動報價及 Aurora 客服繼續共用同一份價格資料。
- WhatsApp 與 LINE 使用同一份報價摘要，避免金額或選項不一致。
- Gemini 只能使用正式報價結果，不得自行編造價格。
- 現有登入保護、使用次數限制、資料版本檢查及金鑰保護保持不變。
- 不在前台或訊息連結中加入任何密碼、金鑰或管理資料。

## 五、測試與驗收

完成後必須驗證：

- 後台可以登入、修改、儲存及重新讀取價格、時間、公告與服務狀態。
- 前台在後台儲存後讀取最新資料。
- 未設定價格的服務仍只顯示「待人工確認」。
- WhatsApp 能帶入正確、完整的報價內容。
- LINE 能複製同一份報價、開啟正確加好友頁面並顯示貼上提示。
- LINE 複製失敗時有可操作的替代提示。
- 顧客可見的預設繁體中文及後台文字不再殘留粵語口語或簡體中文。
- Aurora 客服仍能理解粵語輸入，但以繁體書面語回答。
- 電腦及手機版按鈕容易點擊、排列正常，沒有橫向溢出。
- 現有價格計算、WhatsApp 摘要、Gemini 客服及其他網站功能沒有退步。
- 自動測試、文字檢查及正式版本建置全部通過。

## 六、發布安排

所有檢查通過後，更新現有 Aurora 正式網站及後端。發布後再以正式網址驗證後台、報價、WhatsApp、LINE、手機版及 Aurora 客服狀態。

本次發布不處理 SEO 或獨立網域。
