# Aurora AI 可发现性与 Google 排名设计

日期：2026-07-22

## 目标

让 Aurora Esports Studio 更容易被 Google、ChatGPT 搜寻及其他能够读取公开网页的 AI 工具发现、理解、引用和链接。公开定位统一为：

> Aurora Esports Studio 是服务香港及台湾玩家的线上游戏服务工作室，不设实体门市。

本阶段提高被发现和被理解的机会，但不承诺指定 Google 名次、即时收录或一定被 AI 推荐。

## 已确认现况

- 正式域名、首页、三个游戏服务页、网站地图及机器人规则均可正常以 HTTP 200 读取。
- 一般浏览器、OAI-SearchBot 与 ChatGPT-User 均未被现有机器人规则阻挡。
- ChatGPT 的直接网页读取工具仍可能对正式域名返回内部错误；这与页面本身是否正常打开是两个问题。
- 网站上线时间很短，外部网站对 Aurora 的提及、链接、评论及品牌历史明显少于已经营多年的竞争对手。
- 现有三个游戏页已经包含独立标题、说明、FAQ、案例和基本结构化资料，但品牌信任页、营运规则及跨平台品牌关联仍不足。

## 采用方案

采用「AI 读取基础＋对访客有价值的内容＋外部品牌信号」组合方案。不会制作大量只有关键词不同的薄内容页面，也不会隐藏文字、购买垃圾链接或声称未经证实的第一名、零风险等内容。

### 1. AI 与搜寻机器人读取基础

- 在 `robots.txt` 明确允许 `OAI-SearchBot`、`ChatGPT-User`、Googlebot 及一般搜寻机器人读取公开内容。
- 继续禁止公开抓取后台和 API 路径。
- 保持所有重要页面为不执行 JavaScript 也能读取主要文字的静态 HTML。
- 检查每页的 HTTP 状态、标题、说明、canonical、语言、robots 指令和分享预览资料。
- 更新 sitemap，加入本阶段新增的公开页面，并填写真实更新时间。
- 加入精简的 `llms.txt`，只列出公开品牌说明与主要服务页。此文件属于实验性辅助，不作为 ChatGPT 收录保证。

### 2. 品牌与信任内容

新增两个人类访客也真正有用的页面：

1. `关于 Aurora`：品牌定位、服务地区、三款游戏、服务方式、不设门市、正式联系方式及社交平台。
2. `服务流程与安全说明`：查询、报价、确认、安排、改期、私隐、账号资料、付款资料及人工确认原则。

页面全部使用繁体书面语，不使用广东话口语。不会公开实体地址，也不会虚构公司注册、奖项、顾客数量、胜率或营业年资。

现有三个游戏页会加入通往上述页面的自然内部链接，并强化：

- 可见的服务范围与适用玩家。
- 真实案例及图片说明。
- 报价如何产生、哪些项目需要人工确认。
- WhatsApp 与 LINE 的清楚联络方式。
- 相关游戏页之间的合理导航。

### 3. 机器可理解的品牌资料

- 在全站使用一致的 Aurora Esports Studio 名称、网址和介绍。
- 以 `Organization`、`WebSite`、`Service`、`FAQPage` 和 `BreadcrumbList` 等合适资料描述公开内容。
- 不使用需要实体门市地址的虚假本地商家资料。
- 只把页面上真实可见的内容写入结构化资料。
- 使用 `sameAs` 关联已确认的官方社交平台链接；未知或未核实的账号不会加入。

### 4. Google 内容与排名基础

- 首页明确说明 Aurora 服务香港及台湾玩家，而不是重复堆砌关键词。
- 每个游戏页只针对该游戏的真实段位、服务和案例，避免三页内容互相复制。
- 標題及主標題自然包含用戶會使用的詞，例如「香港傳說對決代打」「傳說對決陪玩」「王者榮耀國服服務」及「HOK 國際服服務」。
- 不为常见错字大量建立页面；错字可在正常问答或同义词说明中自然处理。
- 保留和扩充真实案例，不复制竞争对手内容。
- 网站地图更新后通过 Google Search Console 请求重新抓取主要更新页面。

### 5. 外部品牌信号

网站内会准备一段统一的 Aurora 官方介绍，供 Instagram、Carousell、LINE、Discord 等公开简介使用。所有公开账号应使用同一个品牌名称、正式域名和服务定位。

本阶段不会自动登入或修改外部社交账号，也不会购买链接、制造假评论或发布论坛垃圾内容。可另外交付一份平台简介文案及人工更新清单。

### 6. 监测与后续判断

- 记录来自 ChatGPT、Google 及其他来源的访问。
- 在 Google Search Console 观察曝光、点击、实际查询字词与页面表现。
- 以 30 天为第一轮观察周期，按真实查询数据决定下一批内容，不盲目大量扩页。
- 若公开页面持续能以一般工具读取，但 ChatGPT 直接打开仍稳定失败，再把更换静态网站托管或增加边缘代理列为第二阶段；不会在没有证据时立即迁移。

## 错误处理与安全

- 后台、顾客对话、报价 API、环境变量及密钥不会被写进 sitemap、`llms.txt` 或公开品牌资料。
- 新增页面不会收集额外敏感资料。
- 无法核实的价格、完成时间和特殊结果继续显示人工确认，不由宣传页面自行生成。
- 公开说明会提醒顾客不要发送密码、验证码、付款资料或身份证明给 AI 客服。

## 验收标准

1. 首页、三个游戏页、关于页、服务流程页、robots.txt、sitemap.xml 与 llms.txt 均返回 HTTP 200。
2. OAI-SearchBot、ChatGPT-User、Googlebot 与一般浏览器均能读取公开页面主要文字。
3. 后台与 API 路径仍不对搜寻机器人开放。
4. 每个公开页面拥有唯一标题、说明、canonical 和可见主标题。
5. 结构化资料通过本地测试，内容与页面可见文字一致。
6. sitemap 只包含正式可公开索引的 canonical 页面。
7. 手机及电脑页面没有新增横向滚动、遮挡或明显速度倒退。
8. 现有报价、AI 客服、后台、WhatsApp、LINE 与价格规则不被改变。
9. Tests、Lint 与 Production Build 全部通过。
10. 正式部署后重新验证 HTTP、机器人读取和 Search Console 提交状态。

## 不属于本阶段

- 保证 Google 第一名或保证 ChatGPT 推荐。
- Google Ads 广告投放及广告客户验证。
- 自动付款、实体地址或 Google 商家门市资料。
- 购买外链、假评论、竞争对手内容复制或其他操控排名方式。
